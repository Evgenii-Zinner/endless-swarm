## 2024-05-24 - Canvas 2D Physics Micro-Optimization

**Learning:** `Math.hypot` is often significantly slower than `Math.sqrt(dx*dx + dy*dy)` in hot game loops due to argument handling and overflow protections in JS engines. Avoiding square roots entirely using squared distance comparisons (`dSq < threshold * threshold`) yields even better performance for distance checks. Also, when converting `d < threshold` to squared comparisons, ensure `threshold > 0` is checked first because squaring a negative threshold makes it positive, which changes the logic.
**Action:** When optimizing physics loops or frequent distance checks, replace `Math.hypot` with `Math.sqrt(a*a + b*b)` or use squared distances for comparisons. Always add a positive threshold check when converting a distance comparison to a squared distance comparison.
## 2026-06-29 - Hoisting Expensive Math Functions in Game Loops
**Learning:** Found that `Math.pow()` and `Math.log2()` were being evaluated per-object inside a 250+ item physics loop running at 60 FPS. `Math.pow(0.9, timeScale)` is invariant per frame, and the tier calculation `Math.floor(Math.log2(Math.max(1, radius / 20)))` for garbage collection could be simplified to a single pre-calculated minimum radius threshold outside the loop.
**Action:** Always inspect `.filter` and `.forEach` loops in physics/rendering for math functions that depend solely on frame-constant variables (like `timeScale` or `player.targetRadius`) and hoist them. Convert inverse boundary checks (like checking if `log2(x) < Y`) into direct threshold comparisons (`x < 2^Y`).
